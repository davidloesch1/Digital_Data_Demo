FullStory + Nexus

Visitor segmentation

Set a stable visitor ID and optional user variables for FullStory. The same ID is stored on each behavioral capture as nexus_user_key, so you can cluster by visitor in the Nexus dashboard (Per visitor cloud mode) and build segments in FullStory from the properties below.

Configuration

Values are saved in this browser only (localStorage). Use an opaque ID (e.g. internal customer id or study participant code)—avoid collecting unnecessary PII. FullStory receives FS.identify(visitorId, properties) when the snippet is loaded.

Sent to FullStory as user property nexus_cohort for easy segment rules.

Additional FullStory user properties

Optional key/value pairs (snake_case or camelCase). Empty rows are ignored.

Where this applies

  • Nexus: Pages that load this helper attach nexus_user_key to kinetic payloads (same pattern on your properties after snippet install).
  • Dashboard: Choose Cloud points → Per visitor (mean) to aggregate fingerprints by this ID.
  • FullStory: Define segments or metrics on your user variables (e.g. nexus_cohort) alongside standard FS dimensions.
